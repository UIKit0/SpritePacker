Spritepacker

Small utility for converting gifs and image sequences into sprite atlases.

View project onGitHub

It is this

I created SpritePacker to be able to easily convert GIF animations to PNG sprite atlases to easily import and use them in Unity.

Download

Please, find all SpritePacker releases here.

Features

  • Sprite atlas from a single GIFs
  • Compound sprite from multiple GIFs
  • Keep the folder structure.

Planned features

  • Add a single image to the sprite atlas (now possible with a single-frame gif only).
  • Keep the ordering of the GIFs in the sprite atlas (now only keeps the files sorted alphabetically)
  • Allow use of image sequences.
  • SpritePacker does not currently support combining various frame sizes in a compound sprite atlas. (The size of the frame for resulting atlas is taken from the very first processed animation).
  • Mac/Linux (just needs compiling for these platforms)

Should you have any more suggestions or will find any bugs, please enter them to the issue tracker..

How it works

Create a folder where you will export GIFs, images and image sequences from your favorite editor. For example:

    Sprites
        Barry
            Barry-Run.gif
            Barry-Idle.gif
            Barry-Attack.compound
                Barry-Attack-Mace.gif
                Barry-Attack-Sword.gif
                Barry-Attack-Chair.gif

Run the SpritePacker and pass it the root of the Sprites directory: SpritePacker.exe Sprites. SpritePacker will generate a subdirectory called .package where the result will be generated. Note: Files and repositories prefixed with . (dot) are ignored, so the .package directory will be ignored at the next run.

    Sprites
        .package
            Barry
                Barry-Run.png
                Barry-Idle.png
                Barry-Attack.png

SpritePacker generated Barry-Run.png and Barry-Idle.png from the single GIFs (Barry-Run.gif and Barry-Idle.gif respectively), but made one large PNG file from all the attacks. That's because those were stored in a directory with .compound suffix.

SpritePacker also kept the directory structure the same as was the input.
